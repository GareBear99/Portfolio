# Sponsored Task Workflow

This repo accepts public, issue-based sponsored requests for scoped technical work. Micro-sponsor tasks start at **$1+** when the task is clear and limited.

## Workflow

1. Open an issue using the sponsored task template.
2. Describe the request clearly.
3. Include links, files, repo paths, screenshots, or expected output.
4. Choose whether you want advice, review, documentation, code guidance, or implementation.
5. Add the sponsor/support amount or intended budget.
6. I reply with accepted, declined, or needs-scope.

## Accepted task types

- Repo review
- README cleanup
- Documentation pass
- Plugin feedback
- Bug investigation
- Architecture planning
- GitHub release/submission prep
- SEO/public-facing polish
- Small implementation guidance

## Micro-sponsor policy

A $1+ micro-sponsor request may receive one of the following:

- one concise recommendation
- one short review
- one issue comment
- one small copy edit
- one technical direction/checklist
- one plugin or repo first impression

It does not include full implementation, extensive debugging, private system access, or large multi-file work unless separately agreed.

## Request quality checklist

Good requests include:

- the exact repo/file/page involved
- what is broken or unclear
- what should be true when done
- any deadline or constraint
- sponsor amount or budget range

Bad requests are vague, urgent without context, abusive, or ask for outcomes controlled by third parties.
