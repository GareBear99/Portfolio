## Coverage rule going forward

- Every major repo should have either a project page, a row in this index, or both.
- If a repo is a proof-of-work artifact but not production-ready, label it as a scaffold, handoff, concept, or archive instead of overselling it.
- If a repo is private or not ready for public traffic, keep the project page focused on capability, architecture, and role without implying a live public product.
- For paid/sponsored work, link prospects to the closest proof category: audio/plugin, AI/runtime, simulation/game, finance/ops, or docs/repo cleanup.
