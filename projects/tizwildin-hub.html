What it is

Public source-of-truth spine for the GareBear99 / TizWildin ecosystem: free audio plugin/product routing, sample-pack discovery, itch inventory, release proof, JSON/LLM indexes, sitemap-backed crawl paths, ARC source-spine links, and HUB backend/licensing docs. This portfolio page exists to translate the live HUB/source route into recruiter-readable proof instead of leaving the work buried in one external index.

Portfolio sync fix

The older portfolio copy described this as a narrow desktop plugin manager. The live HUB now acts as the ecosystem front door: public proof routing, plugin discovery, game-asset inventory, music/release proof, ARC source-spine links, and crawlable machine-readable indexes.

Live HUB surface

  • Audio product front door: routes FreeEQ8, FreeVox8, plugin tools, sample packs, and public audio-development resources.
  • Machine-readable proof layer: exposes public-index JSON, llms.txt, sitemap routes, source/repo indexes, and public link graph routes for search engines and AI tooling.
  • Game asset store bridge: routes the GareBearProductionz itch inventory into buyer-facing categories instead of hiding products in a separate storefront.
  • Release and music routing: connects TizWildin all-links, SoundCloud/YouTube surfaces, Release Vault proof, and deconstructed-loop/sample-pack routes.
  • ARC source spine: connects ARC-Core, ARC-Neuron, Arc-RAR, cleanroom runtime, source search, readiness reports, and branch governance routes.

Backend / licensing layer

The HUB also documents a Cloudflare Worker backend for authentication, HUB accounts, community stats, ProEQ8 license activation/deactivation/verification, Stripe checkout, and admin endpoints. In the portfolio, this should be presented as a documented product/backend route, not just a vague “billing integration” claim.

Why it belongs in the portfolio

The HUB is the public distribution layer. The portfolio is the credibility layer. Linking them clearly lets reviewers move from “who is Gary / TizWildin?” to concrete routes, repositories, index files, API docs, release proof, and purchasable/downloadable creator products.
