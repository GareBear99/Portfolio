# Issue Workflow

Issues are the public intake system for sponsored tasks, repo reviews, plugin feedback, hire requests, and general questions.

## Labels

Suggested labels for this repo:

- `sponsored-task`
- `micro-task`
- `repo-review`
- `audio-plugin`
- `documentation`
- `bug-investigation`
- `architecture`
- `seo`
- `needs-scope`
- `needs-payment`
- `accepted`
- `declined`
- `completed`

## Status meanings

- `needs-scope`: the request is too broad or missing details.
- `accepted`: I agreed to the scoped task.
- `needs-payment`: sponsor/budget details are needed before prioritizing.
- `completed`: the response, review, or deliverable has been posted.
- `declined`: not a fit, unsafe, too broad, or outside current availability.

## Public reply templates

### Accepted

```md
Thanks — this is clear enough to scope.

Accepted as a sponsored task. I will treat the expected result as:

- [scope item 1]
- [scope item 2]

If the scope expands, I will split that into a follow-up task.
```

### Needs scope

```md
Thanks for the request. I can help, but this needs a tighter scope first.

Please clarify:

1. Exact file/repo/page
2. Expected result
3. Whether you want advice, copy, review, or implementation
4. Sponsor/budget amount if this is priority work
```

### Declined

```md
Thanks for the request. I am going to decline this one because it is outside the current scope I can responsibly take on.

Good alternatives:

- narrow it to one file or one question
- request a review instead of implementation
- split it into smaller tasks
```
