🤖 ARC AI Operator — live intake

Request a code review

This form POSTs to the ARC AI Operator Cloudflare Worker, which fires a repository_dispatch at gh-ai-operator. The operator clones the target, runs a language-aware heuristic pass, emits a supervised training record for ARC-Neuron-LLMBuilder, and (when the Portfolio issue is supplied) comments the verdict back on that issue.

No server-side code runs on this page. All of the intake logic is a static form + a single fetch() call to the Worker.

The deployed Cloudflare Worker. Change only if you're running a fork.
Public repositories and public PRs only. Private targets must be reviewed via email.
If set, the operator will comment its verdict on this issue. Must look like owner/repo#NN.
Sent as the x-arc-token header. This page does not persist it; it lives only in this browser tab.

What happens next

  1. The Worker validates your target URL and (optional) portfolio_issue.
  2. It calls GitHub's repository_dispatch endpoint at GareBear99/gh-ai-operator with event type code-review-request.
  3. The ai-review-dispatch.yml workflow runs review_target.py against the target in a throwaway clone.
  4. Heuristic + language-aware rule packs produce a verdict. When Cloudflare Workers AI credentials are set, an LLM layer adds semantic critique.
  5. If portfolio_issue was supplied, the verdict is commented there. A llmbuilder-training-export artifact is uploaded for the nightly LLMBuilder ingest.

The Worker never runs the model itself; it is a thin authenticated dispatcher. Attack surface is bounded to github.com / gist.github.com targets.
